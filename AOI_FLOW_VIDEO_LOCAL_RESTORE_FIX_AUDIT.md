# AOI FLOW 動画UI・LOCAL選択・既存確認動画復元 監査

## 修正対象
- `app/flow/drafts/new/components/ProductVideoPanel.tsx`
- `components/video/NonAiVideoActions.tsx`

## 実施内容
1. LOCAL Render ボタン
   - localhost / 開発環境で押せない状態を避けるため、UI上は常時選択可能に変更。
   - ffmpeg未導入時はMP4変換エラーで止まらず、WEBM保存成功として扱う既存退避導線を維持。

2. 複数画像選択
   - アップロード済み商品写真リストから、行クリック/丸ボタンで追加・解除できる構造を維持。
   - 選択枚数と順番表示を維持。

3. 不要UI整理
   - 商品画像側の追加アップロード用「ファイルを選択」は出さない構造を維持。
   - 動画素材の状態テキストブロックは復活させず、画面圧迫を避けた。

4. 既存機能復元
   - 監査で削除リスクになっていた「クロマキー確認動画」を復元。
   - 監査で削除リスクになっていた「代表動画確認」を復元。
   - ただし構造破壊を避けるため、動画表示は `maxHeight: 220` のコンパクト表示に制限。

5. 画像巨大化対策
   - アップロード画像選択欄は小型サムネイルリストのまま維持。
   - 商品画像側の大画像プレビューは追加していない。

## 監査結果
- `npx tsc --noEmit`：成功
- `npm run build`：コンパイル成功後、TypeScript工程でタイムアウト
  - 表示された範囲ではコンパイルエラーなし
  - タイムアウト理由はNext buildの最適化/型工程が長いため

## 注意
- Local RenderでMP4を作るにはMac側に `ffmpeg` が必要。
- ffmpegが無い場合でもWEBMは保存されるため、ユーザー向けには「失敗」ではなく「WEBM保存成功、MP4化は未実行」と表示する設計。
